# Manuscript Guidance

Canonical guidance for manuscript, Markdown, and Quarto-style narrative work in
this repository.

## Scope

Apply this file to:

- `Manuscript/**/*.md`
- `Manuscript/**/*.qmd` if Quarto files are added later
- narrative documentation that embeds analysis results

PastHumanImpact currently keeps manuscript planning and review material under
`Manuscript/_internal/`. There is no repository-level Quarto website scaffold in
the current tree, so do not invent website paths or `docs/` output unless the
user asks for that work explicitly.

## Prose Style

- Keep scientific prose precise and conservative.
- Prefer one sentence per source line in Markdown or Quarto files.
- Do not wrap prose at 80 characters; the 80-character limit applies to R
  source code, not manuscript text.
- Avoid hardcoded analysis-derived numbers in manuscript prose when an
  executable inline result or generated table can be used.

## Analysis Values in Text

When a number comes from data, models, or configuration, keep it connected to
the analysis source. In `.qmd` files, use inline R expressions. In plain
Markdown, include enough nearby context to trace the value back to the relevant
script, table, or output file.

Stable external constants and publication identifiers may be literal.

## R Code Chunks

All R code in `.qmd` chunks follows `.ai/r-coding.md`.

Use a setup chunk that anchors paths and loads project configuration:

```r
#| label: setup
#| include: false
here::i_am("Manuscript/<file>.qmd")

source(
  here::here("R/00_Config_file.R")
)
```

Use chunk labels that describe the computed result. Suppress messages and
warnings unless the output is intentionally part of the document.

## Reading Targets Outputs

Target stores are under the external data storage path resolved by
`R/00_Config_file.R`. Build store paths explicitly:

```r
set_store <-
  file.path(
    data_storage_path,
    "Targets_data",
    "analyses_h1"
  )

targets::tar_read(
  name = target_name,
  store = set_store
)
```

Use the store that matches the pipeline:

| Pipeline script | Store |
| --- | --- |
| `R/target_pipelines/01_pipeline_pollen_data.R` | `Targets_data/pipeline_pollen_data` |
| `R/target_pipelines/02_pipeline_paps.R` | `Targets_data/pipeline_paps` |
| `R/target_pipelines/03_pipeline_events.R` | `Targets_data/pipeline_events` |
| `R/target_pipelines/04_pipeline_predictors.R` | `Targets_data/pipeline_predictors` |
| `R/target_pipelines/05_pipeline_hvar_spatial_temporal.R` | `Targets_data/analyses_h1` |
| `R/target_pipelines/06_pipeline_multidimensional_shifts.R` | `Targets_data/analyses_h2` |

## Figures

Figures are generated by scripts under `R/visualisations/` and written to the
project's configured output locations. Do not manually edit generated figures or
tables unless the task is explicitly about manuscript-only annotation.

When adding new figure code:

- source `R/00_Config_file.R`
- use `here::here()` or `file.path(data_storage_path, ...)` for paths
- keep dimensions and style consistent with `R/00_Config_file.R`
- save outputs under `Outputs/` or the configured external data path
